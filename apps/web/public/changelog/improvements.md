# Improvements

## Meetings list view
The Meetings page now stays focused on the list, while full meeting details open on a dedicated detail page.

## Meeting page actions
Meeting detail pages now make common actions easier to find, including retrying processing and jumping to the linked deal.

## Cleaner recordings navigation
Meetings and browser recordings now use the same tab and filter styling as the Pipeline pages for a more consistent CRM experience.

## Faster meetings list
Meeting lists now load with a lighter payload, keeping the page more responsive while details load only when opened.

## Proposal workspace polish
Proposal list and detail pages now have cleaner layouts, clearer version handling, and fewer inactive navigation distractions.

## Local changelog content
What's New content now comes from markdown files in the app instead of database rows, making release notes easier to review with code changes.

## Reseller revenue consolidation
The old standalone reseller revenue page was removed after its important views moved into the main Revenue page.

## Shared pipeline stage pills
Revenue status cells now use the shared pipeline stage pill styling so stage labels stay visually consistent across the app.
